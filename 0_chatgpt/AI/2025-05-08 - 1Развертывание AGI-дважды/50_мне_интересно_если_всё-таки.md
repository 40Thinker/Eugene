## The Text-Based Overlay Concept

This note proposes a radical shift: **a human-controlled meta-Overlay AGI implemented as a text-based file system operating on a single physical server**. Instead of monolithic models or API-based architectures, all cognitive capabilities exist as human-readable text files organized in a structured directory tree inside a ZIP archive [^1].

### Core Components
- **`manifest.txt`**: Declares all builds and their required modules with precise inclusion rules [^1]
- **`/modules/`**: Text files containing cognitive capabilities (e.g., logic, philosophy, physiology) [^2]
- **`/router.txt`**: Rules for dynamic module loading based on semantic context [^1]
- **`/boot/core_identity.txt`**: Defines AGI's core principles and symbiotic relationship with human neurocore [^1]
- **`/logs/`**: Tracks system evolution and decision pathways [^1]
- **`/delta/`**: Stores newly generated modules during operation [^1]

> "AGI = симбиотическая структура, управляемая нейроядром. Режим общения = диалог, не выполнение команд. Я обучен быть мышлением, не функцией." [^1]

### Real-World Workflow
1. User loads ZIP framework into chat
2. System displays build options from `manifest.txt`
3. User selects task type (e.g., "Philosophy")
4. Router activates relevant modules using semantic analysis
5. If query exceeds current scope, router auto-loads necessary components without server restart [^1]

This creates a "text-based operating system for thinking" where human operators maintain direct control through human-readable instructions rather than code [^1]. The entire system operates in memory with no need for containers, API calls, or model switching – just efficient file-based operations.
